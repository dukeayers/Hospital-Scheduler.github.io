Project Overview

We are currently tasked with creating a way to effectively and efficiently help the local hospital, Flagstaff Medical Center, to automate their scheduling process. This consists of weekly meetings with our NAU mentor and bi-weekly meetings with the FMC staff. On this website you will find our prototype descriptions, information about our team, and a link to go to our wiki. The wiki contains all of our paperwork information which includes the requirements document, use cases, and much more.

Prototype One

General Description:

     Our first prototype was primarily focused on getting our web technologies up and running. This meant that we needed to ensure that AngularJS, PHP, and Bootstrap were going to work well with one another with as minimal problems possible. We all took the time to delve into the Google Calendar API and pair it with a plugin that would output the data in a nice format. This meant that we needed to use OAuth to interact with the API to send/receive data (we made it possible to create an event and insert it into the calendar which is shown in Photo 2).

    Another key part to this prototype was the use of AJAX to grab the data (display it into a calendar), and send data to the API (insert an event into the calendar). Finally, we are using PHP, which means that we have a MySQL DB set up on our backend. We placed some example data into the database and used PHP to make the call to the database. AngularJS played a big part in calling that PHP file that sent the request for the information to the database - this feature allows us to have a single page application. This prototype will be stored for later, for now we are going to be focusing on creating prototype 2. If you would like a copy of our prototype, please visit our Wiki and look for instructions there.
